Every environment that ships with NeMo Gym lives in [`resources_servers/`](https://github.com/NVIDIA-NeMo/Gym/tree/main/resources_servers) and has a config under `resources_servers/<name>/configs/`. This catalog groups them by domain so you can pick one without browsing the directory tree.

<Tip>
Each row links to the resources server's README on GitHub. For the live programmatic listing, run `ng_list_benchmarks` in your shell — it scans the repo and prints every config it finds.
</Tip>

## Status Legend

The `verified:` field in each YAML config tracks whether the team has baselined and reviewed the benchmark.

| Status | Meaning |
|---|---|
| **Verified** | `verified: true` — baselined by the team, expected output documented, reward profile reviewed. Safe to use as a published-number reference. |
| **Provisional** | `verified: false` — runnable, but expected output is not locked. Fine for development; cross-check before publishing numbers. |
| **External wrapper** | Wraps a third-party benchmark library (Aviary, Reasoning Gym, OpenEnv, VLMEvalKit). Status mirrors the upstream library; verify in their docs. |

Most shipped benchmarks today are `verified: false`. That's not a quality signal — it tracks formal baselining, not whether the benchmark works. The team is actively raising coverage; new entries to the catalog start as Provisional and graduate as baselines are locked.

## How to Add a Benchmark to This Catalog

The pre-commit hook `update-readme-table` auto-adds new YAML configs to the root `README.md` table. This catalog page is curated separately to group entries by domain and call out external wrappers — when you add a new benchmark, add a row here too.

For the full author flow — resources server, verifier, dataset, baselining — refer to:

- [Add a Benchmark](/latest/evaluate/add-a-benchmark) — short eval-side flow.
- [Adding a Benchmark](/latest/build-environments/integrations/add-a-benchmark) — full author tutorial.

A new benchmark earns the `Verified` flag once it has a baselined expected output, a documented validation recipe (refer to [Validate Your Setup](/latest/evaluate/validate-your-setup)), and team review.
